- [ ] **Step 4: Write `## The compilation pipeline`**

Content beats. Use `### H3` subsections here (one of three sections allowed):

- `### JIT is off the table on Wasm` — sljit doesn't list Wasm as a target; Wasm forbids RWX memory pages. Build with `--disable-jit`. `setjmp`/`longjmp` becomes a non-issue once JIT is off.
- `### The performance penalty` — JIT-off slowdown is **3–10× typical, 15–25× worst case** on backtracking-heavy patterns. Cite the OpenResty / GNOME GtkSourceView / pg_jitter measurements as the source of those bands (do not claim to have run them yourself).
- `### Toolchain choice` — wasi-sdk vs Emscripten. wasi-sdk is leaner; Emscripten gives you JS shim code you don't need.
- `### Binary size and boundary cost` — Wasm output is **~400–700 kB; 150–250 kB of that is Unicode tables**. The Go↔Wasm boundary adds **50–100× overhead on tiny inputs**, so batching is mandatory.

- [ ] **Step 5: Write `## Prior art: one stale experiment, and a telling counter-example`**

Content beats — this is the revealed-preference argument:

- `bobby-stripe/go-pcre`: last tag **v1.0.1 (Oct 2021)**, ships **PCRE2 10.38**, **zero importers on pkg.go.dev**. The one experiment that exists, and it has been stale for over four years.
- OWASP Coraza v2 had an optional cgo PCRE plugin (`coraza-pcre`); v3 removed it and is RE2-only. Anuraag Agrawal's CNCF talk justifies the move.
- `wasilibs` ships `go-re2`, `go-aho-corasick`, `go-pgquery`, `go-yamllint`, and more — but no `go-pcre2`. The team that knows how to do this exact thing did not do it.

Close the section with one explicit sentence: that pattern is the most important data point in this whole analysis.
